终端

有 nnn 个主机（可能是物理的，也可能是虚拟的），终端就是一块屏幕。打开一个终端就是一个小黑框，这个终端和某个主机“连接了一根线”。如果打开多个终端，意思是打开了多个屏幕，这些终端可以连接到同一个主机上，在一个终端中对主机做一些修改，那么另外一个连接到同一个主机的终端上也是可以看到这个改变的。

终端是联网的，可以连接到任何一个服务器（？）。比如使用ssh登录。

关掉终端只是把屏幕关掉了，而主机是不会关掉的。但是关掉终端之后，你正在运行的程序会停止运行。

如果使用的是tmux，那么退出之后程序也不会关掉。比如登录到你的服务器之后打开tmux，在里面下载docker，那么你关闭终端不会影响下载过程。

所以只要有一个终端就可以了，我们只要学会如何使用终端，即学习终端中的各种命令，就能操作各种服务器。

服务器向终端发送信息会占用带宽，如果是好多人用一个服务器且没有设置带宽限制的话会感觉到。

Linux文件系统概述

根目录的名字为/ ，/ 就是一个文件夹。

/ 文件夹下面可能会有 bin （里面放一些可执行文件，比如一些指令就是在这个里面，我们敲命令其实就是执行 bin 下面的对应名字的可执行文件），etc ，lib，boot，home（用户的家目录），proc 等各种各样的文件夹，acs acwing server

登录之后看到的目录是 home 目录下的一个目录，如果这个服务器上不只有你一个用户，那么不同的用户在 home 的目录下有对应的目录名字。

Linux下描述路径：

  • 绝对路径，从 / 开始描述
  • 相对路径，不以 / 开头就是相对路径。. 表示当前目录，.. 表示上一级目录。

~ 为家目录，是相对某个用户来说，比如 /home/zcx 就是 zcx 的家目录。

ssh

ssh登录

背景

未来工作的时候我们可能会用到多个服务器，比如有管web的服务器，有管评测机的服务器等。有些时候我们需要到这些服务器上做一些开发和维护，这个时候我们就可以使用ssh进行服务器的免密登录（需要先配置一下）。

user hostname password，租用服务器之后一般会获得这三个信息。

hostname 一般来说是一个 ip 地址。

登录服务器

使用命令 ssh user@hostname 即可实现登录。如果是第一次登陆的话，可能会提醒我们从来没有登陆过这个服务器，这是因为我们从来没见过这个服务器的 ip 地址的哈希值（如果之前登陆过的话，那么这个哈希值会存储到 ~/.ssh/known_hosts 这个文件里面）。

如果没有配置免密码登录，那么每次登录的时候都需要使用密码才可以登录。

完成之后，我们就进入了一个全新的服务器。

使用 exit 或者 ctrl + d 命令可以退出这个服务器。

ssh 登录的时候使用的端口默认为 22 ，可以用 -p number 来指定端口号。

配置文件

如果感觉每次登录的时候需要敲 user@hostname 太长了且背不下来，那么可以创建一个配置文件，给这个服务器创建一个别名。

配置文件名叫 config ，需要放在 ~/.ssh/ 目录下。config 配置文件的内部的信息的格式是：

Host myserver1
	HostName IP地址或者域名
	User 用户名
	
Host myserver2
	HostName IP地址或者域名
	User 用户名

使用密钥免密登录

ssh 使用的是 rsa 密钥。

如果之前没有生成过密钥，那么需要使用命令 ssh-keygen 命令生成密钥和公钥，都放在 ~/.ssh 目录下。其中 id_rsa 是密钥，id_rsa.pub 是公钥。

如果想免密登录哪个服务器，那么只需要把这个公钥复制到那台服务器的 ~/.ssh 目录下的 authorized_keys 文件中。

这样就可以直接通过 ssh server_name 登录到对应的服务器了。

scp传文件

命令格式 scp src dst ，把 src 位置的文件复制到 dst 里面。

复制文件夹的话，可以使用 -r 参数，但是要放到 scp 后面紧接着。

src 如果是本地的，那只需要写正常的路径就好了。

dst 如果是其他服务器，那就特殊一点，需要先加上 server_name: ，然后再跟上这个 server 内的地址才可以。

比如 scp -r ~/tmp server1:~/ ，意思就是把我现在所在的服务器上的 ~/tmp 目录传到 server1 上的 ~/ 目录下。

src 是远程服务器，dst 是本地服务器同理。

如果 src 和 dst 都不是本地，可以先把 src 拷贝到本地，然后再拷贝到 dst 里面。直接拷贝虽然也可以，但是需要有一些权限。

scp传文件的一个重要应用就是把远程服务器的开发环境做一个配置，比如把 vim 和 tmux 的配置文件拷贝上去。

比如 scp ~/.vimrc server1:~/ 就可以把本地的 .vimrc 拷贝到 server1 服务器上。

免密登录

这部分在 ssh 中讲过。

然后回到本机，给服务器起一个别名，放到 .ssh/config 里面，以后可以直接别名登录。
